# Задание 3: Архитектура отправки push-уведомлений

## Описание
В мобильное приложение интернет-магазина "Петрушка Зеленая" нужно добавить получение push-уведомлений:
- о забытой корзине
- об отмене заказа
- рекламные рассылки

На бэкенде — микросервисная архитектура.

## Решение

Архитектура построена с использованием **C4 model** (уровни Context и Container).

### Что показывает
Взаимодействие нашей системы с внешними системами и пользователем.

|-----|--------------|----------|----------|
| Person | Пользователь | — | Человек, который делает заказы и получает push-уведомления |
| Software System | Петрушка Зеленая | Петрушка Зеленая | Интернет-магазин (бэкенд + мобильное приложение) |

## 2. C4 Level 2 — Container Diagram (диаграмма контейнеров)

### Что показывает
Внутреннее устройство системы "Петрушка Зеленая": микросервисы, очереди, базы данных.

### Контейнеры 

| № | Контейнер | Технология | Описание |
|---|-----------|------------|----------|
| 1 | Мобильное приложение | Swift / Kotlin | Получает токен, отправляет в бэкенд, отображает пуши |
| 2 | API Gateway | Node.js / Express | Принимает запросы от мобильного приложения |
| 3 | Сервис корзины | Java / Spring Boot | Публикует событие `cart_abandoned` |
| 4 | Сервис заказов | Java / Spring Boot | Публикует событие `order_cancelled` |
| 5 | Рекламный сервис | Python / FastAPI | Публикует событие `promo_push` |
| 6 | Message Broker | RabbitMQ | Очередь сообщений (буферизация событий) |
| 7 | Notification Service | Node.js / Python | Формирует и отправляет push-уведомления |
| 8 | Device Token DB | Redis | Хранит push-токены пользователей |

## Ключевые архитектурные решения

| Решение | Почему |
|---------|--------|
| Отдельный Notification Service | Чтобы не смешивать логику отправки пушей с бизнес-логикой |
| Message Broker (очередь) | Гарантирует доставку событий, даже если Notification Service временно недоступен |
| Device Token DB (Redis) | Быстрый доступ к токенам пользователей |
| gRPC между микросервисами | Быстрее и компактнее, чем REST, для внутреннего общения |
| HTTPS / REST между приложением и API Gateway | Стандарт для клиент-серверного взаимодействия |
| APNs и FCM как внешние системы | Только Apple и Google имеют доступ к доставке пушей на устройства |
